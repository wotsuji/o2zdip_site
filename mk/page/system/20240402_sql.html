SQL 学びなおしのメモ

最初に関わったシステムと呼べるレベルのモノというと私はAccessになる。
特にVBAよりもSQLとフォームを駆使していたと思う。
システム設計をしているとSQLはメイン言語（PHPやPythonなど）に成り得ないため
あくまでシステムを構成するサブ言語となり出来て当たり前のように使われている印象がある。
というレベル帯において、改めてSQLを学びなおしてみるメモである。

◆CASE式

CASE式は性別や地域などを数値で持っている場合にSQL内でデータに変換する場合に使ってきているが
最近はプログラム側で変換用の定数をConfigファイルやConfigクラスのような形で一元化したり
Modelクラスで変換するのがベターなのかなと思う。


CASE [カラム名]
    WHEN '1' THEN 'A'
    WHEN '2' THEN 'B'
    WHEN IN ( 3','4') THEN 'C'
    ELSE 'D'
END

テーブルにカラムが用意されていない構造で一時的に変換を行うならSQLでCASE式を使うのはアリよりのアリ。
ELSE句が無いとWHEN句でマッチしなかった場合はNULLとみなされるので注意。特にUPDATEでCASE句を使う場合。

◆「= NULL」ではなく「IS NULL」と書かないといけない

　NULLは比較できない（値でも変数でもない）
　NULLは「値が無い」という文脈でしかない
　空文字とNULLは違うが空文字をNULLとして扱うDBMSがある（ORACLEなど？）
　ORDER BY句の並び順もDBMSによって違いがある

◆値にNULLを含む場合の注意

membger_bのageにnullを含むデータがあった場合に動きが変わるので注意。


　SELECT name,age
　FROM member_a
　WHERE age IN (SELECT age FROM member_b)
　下記のようにIN句にNULLが含まれるケースになる

　WHERE age IN ('20','22',NULL)

◆EXISTS句

　EXISTSは必ずサブクエリを使う
　サブクエリの結果、レコードが存在した場合は「TRUE」、そうでない場合は「FALSE」を返す
　JOINはテーブル結合を返すが、EXISTSはテーブル単体を返すため、JOINより早く動作する。
　NOT EXISTS で結合先が存在しないレコードも返せる


SELECT * FROM member
WHERE EXISTS (SELECT * FROM book  WHERE member.id = book.member_id)
